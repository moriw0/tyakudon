着丼 デザインシステム

base.css を参照するすべてのページが従うべき標準

1. コンセプト

素朴さを最大の強みとする。ブラウザのデフォルトに近い外観を維持し、 CSS は最小限に留める。装飾より情報密度を優先する。 モバイルでは列を減らすことで表形式を維持する。

  • テーブル形式で統一（リスト形式と混在させない）
  • 外部フォント・アイコンライブラリ不使用
  • JavaScript は最低限

8. フォーム

全フォーム要素に width: 100%; box-sizing: border-box; が適用される。幅を個別指定したい場合のみ inline style で上書き。

ラベルとフィールドのレイアウト

label { display: block } が v2.css で適用されているため、<br> なしでラベルが独立行に表示される。
チェックボックス・ラジオボタンの直後の label には input[type="checkbox"] + label, input[type="radio"] + label { display: inline } が効くため inline 表示になる。 

/* v2.css */
label { display: block; }
input[type="checkbox"] + label,
input[type="radio"] + label { display: inline; }

/* ビュー: br 不要 */
<p>
  <%= f.label :email %>
  <%= f.email_field :email %>
</p>

    document.getElementById('overlay').classList.remove('active');
    document.getElementById('modal').style.display = 'none';
    document.body.style.overflow = '';

</script>

14. インラインアクションバー

詳細ページ下部などで、複数のテキストアクションを横並びに配置する場合に使用。&nbsp;|&nbsp; で区切る。

← 戻る  |  いいね（6）  |  シェア

ボタン（.btn）はページの主アクション用。インラインアクションバーはサブアクション（戻る・いいね・シェアなど）に使い、両者を混在させない。

<p>
  <a href="#">← 戻る</a>
  &nbsp;|&nbsp;
  <a href="#">いいね（6）</a>
  &nbsp;|&nbsp;
  <a href="#">シェア</a>
</p>

15. インライン強調（<strong> / <code>）

<strong> — 主要な数値・最重要情報

そのページで最も重要な値に使用する。意味的な重要度の強調であり、見た目の太字化は副次的な効果。1ページで多用しない（1〜2箇所まで）。

ページ使用箇所理由
result待ち時間（例: 5分18秒）記録の主役となる値

<code> — 等幅表示が必要な値

タイマー・時刻など、等幅フォントで視認性を上げたい数値に使用する。スタイル上書きは行わない。

ページ使用箇所理由
measure経過時間（例: 23:47）カウントアップ中の数値。等幅で桁ズレを防ぐ

使い分けまとめ

タグ使う場面使わない場面
<strong>ページの主要な結果値（待ち時間など）装飾目的の太字、複数箇所への多用
<code>タイマー・時刻など等幅が意味を持つ値通常の数値（待ち時間の表示など）
装飾なし上記以外のすべての値—
 
<!-- result: 待ち時間（ページの主役） -->
<td><strong>5分18秒</strong></td>

<!-- measure: 経過時間（等幅表示） -->
<td><code>23:47</code></td>

<!-- 通常の数値（装飾なし） -->
<td>35分</td>

17. Flash メッセージ

notice と alert を両方表示する。

種類表示用途例
notice 通常テキスト（黒） 登録完了、送信完了など
alert 赤字テキスト 不正アクセス、認証エラー

実装

<% if flash[:notice] %><p><%= flash[:notice] %></p><% end %>
<% if flash[:alert] %><p class="flash-alert"><%= flash[:alert] %></p><% end %>
